# # 백준 9093번, Word Reversal

입력으로 문장들이 주어지면 단어들까지만 역순으로 바꿔서 출력하는 문제이다. 또 cin으로 받으려다가 아차 싶어서 getline(cin, s)으로 받고 앞에서 cin으로 받았으니 cin.ignore()은 명시해줘야 한다.

cin.ignore();
	
string s, split;

for (int i = 0; i < t; i++) {
	getline(cin, s);

	istringstream ss(s);

	while (ss >> split) {
		reverse(split.begin(), split.end());
		cout << split << ' ';
	}
	cout << '\n';
}

풀면서 생각이라도 나서 다행이다, 쌩으로 split하려다가 istringstream 생각나서 썼다. 안찾아보게 사용법이라도 숙지하고 있다가 나중에 정리해야겠다.

괄호를 써서 스트림을 받고, 자료형에 따라서 입력받듯이 반복 조건에 >>를 써주면 즈그?알아서 잘 받는다.

반복문 내에서는 <algorithm> 헤더에 있는 reverse로 역순으로 만든다음 출력하면 그만이다.

t = int(input())

for i in range(t):
    a = list(map(str, input().split()))
    [print(b[::-1], end=' ') for b in a]
    print()

이걸 파이썬으로도 풀어보자. C++만 쓰다보니까 pythonic한게 뭔지 모르겠다. 지금 생각하건데 무조건 숏코딩만 하는 것도 아니고 충분히 간결하게 코드를 작성하되 이해하는데 어려움은 없을 정도가 아닐까?

def f(s): return s[::-1]
t = int(input())
a = [list(map(f, input().split())) for i in range(t)]
for b in a: print(*b)

@faper (opens new window)씨 아이디어가 들어간 코드다. list comprehension으로 반복문으로 depth 구문할 거 줄이고, 리스트에 저장하는 과정에서 함수를 정의해서 쓴다. 그리고 마지막 출력에서는 리스트 요소들을 unpacking 하기 위해서 Asterisks(*)를 쓰고있다. 옆에서 이 사람이 더 줄이겠다며 2줄로 만드는 것까지 목격했다!

[print(*b) for b in [list(map(f, input().split())) for i in range(int(input()))]]

# # 백준 17413번, 단어 뒤집기 2

문자열 가지고 노는 문제가 개인적으로 참 까다로운 것 같다. 문제는 단순한데 태그 안에 내용은 그대로 놔두고 다른 건 단어를 뒤집으면 된다. 뭔가 스트림으로 풀릴 것 같았는데 예제 입력 7을 보고 그냥 문자열을 순회하면서 출력에 업데이트 하기로 했다.

&lt;   space   >space space space&lt;    spa   c e>

Less than, 즉 < 인지 검사하고, 맞다면 rword 를 '뒤집을 지'를 결정하는 flag 를 false로 만든다.

if (s[i] == '<') {
	// if rword.size() > 0, then yield
	if (rword.size() > 0) {
		yield(out, rword);
		rword.clear();
	}

	out += s[i];
	flag = false;
	continue;
}

결과적으로 out 를 출력하는 건데 어디서 append 할지는 따라가다보면 알 것이다. 뒤집는 단어 다음에는 아니면 < 가 올 수 밖에 없다는 사실이 중요한 것 같다. 그래서 해당 condition에서 앞에 out에 yield 하는 routine을 넣었다.

if (!flag) {
	out += s[i];
	if (s[i] == '>')
		flag = true;
} else {
	if (!isLowerOrNumber(s[i])) {
		// if meet a space, then yield
		yield(out, rword);
		rword.clear();

		out += s[i]; // never append '<' (only ' ')
		continue;
	}
	rword += s[i];
}

Less than이 등장했다면, greater than이 나올 때까지 결과에 append한다. Greater than이 등장했다면, flag를 true로 만들고 다음 else branch로 갈 수 있게 한다.

Else에서는 rword에 append하고 공백이 등장할 경우 out에 yield하게된다.

if (rword.size() > 0)
	yield(out, rword);

마지막으로 문자열의 끝은 '>' 아니면 뒤집는 단어의 문자가 온다는 것 때문에 rword가 비어있거나 .clear()되지 않았다면 yield 하는 것까지 확인해줘야한다.

void yield(string& out, string& word) {
  reverse(word.begin(), word.end());
  out += word;
}

중복 때문에 함수 정의해서 뺏는데, 참조로 넘겨줘야 업데이트 된다는 점까지 보면 될 것 같다.

# # 백준 10799번, 쇠막대기

괄호 문자열 문제만 보면 거부감이 들었는데, 방법을 생각해 내는 데까지는 얼마 안 걸리는 문제다. 밑에 그림에서 레이저가 막대를 자르는데 몇 조각이 나올까 그런 문제다.

()는 레이저, 그 외 레이저를 만드는 괄호가 아닌 경우 (와 )는 각각 쇠막대기의 시작과 끝점이다.

그러면 당연하게도, 문제 앞에서 설명하는 '쇠막대기는 자기보단 긴 막대 위에만 놓을 수 있고 끝점은 안겹친다, 레이저는 막대의 끝점과 겹치지 않는다'는 사실은 자명하다.

앞뒤 괄호의 3가지 조합만 생각하면 된다.

  1. (). 레이저일 경우, 막대의 현재 층수와 끝났던 막대 개수를 세어 더한다. 다시 끝나는 막대 개수를 세는 변수는 0으로 만든다.

  2. ((. 막대의 현재 층수만 increment한다.

  3. )). 층수를 decrement하고, 끝나는 막대 개수를 increase.

이렇게만 구현하면 뭐가 예제 출력보다 조금 적게 나온다. 그래서 무지성으로 +1 해버리면 틀릴 것이다. 직전의 업데이트 이후부터 현재 업데이트 사이에 대한 끝나는 막대 개수만을 더하게 되는데, loop를 빠져나오려면 레이저가 아닌 무조건 막대의 끝으로 끝나게 된다. 따라서 그만큼의 끝나는 막대 개수가 남게되는데, 그것마저 더해주면 답을 출력하게 된다.

# # 백준 11005번, 진법 변환 2

10진법 수 N을 B진법으로 변환한 수로 출력하면 된다. 단, B는 2이상 36이하의 진법이고 숫자로 표현할 수 없는 것은 10을 A로해서 35은 Z까지 잘 나타내면 된다, 그리고 N는 10억 이하의 자연수이다.

while (n >= powerOfInt(b, p))
	p++;
p--;

while (p >= 0) {
	c = n / powerOfInt(b, p);
	n = n % powerOfInt(b, p);
	p--;

	if (c >= 10) cout << (char)('A' + c - 10);
	else cout << c;
}

이런 문제에서 틀틀맞이 나와버려서 상심이 크지만, 내가 만든 테스트 케이스로 검증하는 과정이 중요하다는 것을 시간이 갈수록 느낀다.

In:
100 8

Out:
144

In:
8 8

Out:
10

In:
1 2

Out:
1

In:
0 2

Out:
0

N를 B진법의 수로 나눠가면서 출력하는 식으로 구현하기 위해서, B진법의 승수를 증가시켜서 P를 찾고 뭐 저렇게 하려고 했는데 첫번째로 >= 대소 관계를 포함시켜야 하는 점이 가장 중요하다. 이게 나누어 떨어지는 경우를 포함시키지 않으면, 2진법인데 2가 나오고 8진법인데 8이 나온다. 그리고 두번째로 최댓값은 항상 생각해야한다. N이 최대 10억이기에, 36진법을 써버리면 Line 1에서 36억에 가까운 값이 나와버리면 int의 최댓값을 넘어버리게 된다.
